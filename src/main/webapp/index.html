Smart Campus API

REST API for managing campus rooms and IoT sensors — built with JAX-RS (Jersey 2.x) on Apache Tomcat 9.

Base URL http://localhost:8080/Smart-Campus-API/api/v1
Data is stored in-memory — resets on every Tomcat restart. Always: 1 POST room → 2 POST sensor → 3 POST reading.

POST/api/v1/rooms Create a room▶

http://localhost:8080/Smart-Campus-API/api/v1/rooms
Postman steps
  1. Method: POST
  2. URL: http://localhost:8080/Smart-Campus-API/api/v1/rooms

Sensors

GET/api/v1/sensors List all sensors (optional ?type= filter)▶
All sensors
http://localhost:8080/Smart-Campus-API/api/v1/sensors
Filtered by type
http://localhost:8080/Smart-Campus-API/api/v1/sensors?type=temperature
Postman steps (with filter)
  1. Method: GET
  2. URL: http://localhost:8080/Smart-Campus-API/api/v1/sensors
  3. Click Params → KEY: type VALUE: temperature
  4. Click Send
200JSON array — all sensors or filtered
POST/api/v1/sensors Register a sensor▶
Full URL
http://localhost:8080/Smart-Campus-API/api/v1/sensors
Postman steps
  1. Method: POST
  2. URL: http://localhost:8080/Smart-Campus-API/api/v1/sensors
  3. Body → raw → JSON
  4. Paste JSON body below → click Send
  5. Room id "1" must exist first — run POST /rooms first
Raw JSON body — ACTIVE sensor
{ "id": "S1", "type": "temperature", "status": "ACTIVE", "currentValue": 0, "roomId": "1" }
Raw JSON body — MAINTENANCE sensor
{ "id": "S3", "type": "temperature", "status": "MAINTENANCE", "currentValue": 0, "roomId": "1" }
201Sensor created and linked to room
422Unprocessable — roomId does not exist
409Conflict — sensor id already exists
400Bad Request — id field missing

http://localhost:8080/Smart-Campus-API/api/v1/sensors/S1/readings

Error case tests

POST/api/v1/sensors Invalid roomId → expect 422▶
Full URL
http://localhost:8080/Smart-Campus-API/api/v1/sensors
Raw JSON body (roomId 999 does not exist)
{ "id": "S2", "type": "humidity", "status": "ACTIVE", "currentValue": 0, "roomId": "999" }

POST/api/v1/sensors/S3/readings MAINTENANCE sensor reading → expect 403▶
Step A — create maintenance sensor first (POST /sensors)
{ "id": "S3", "type": "temperature", "status": "MAINTENANCE", "currentValue": 0, "roomId": "1" }
Step B — POST a reading to S3
http://localhost:8080/Smart-Campus-API/api/v1/sensors/S3/readings
{ "id": "R2", "timestamp": 1713780000, "value": 30.2 }
403Forbidden — sensor in MAINTENANCE cannot accept readings
